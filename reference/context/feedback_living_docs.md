On RealShare, keep **all** the workspace markdown docs continuously updated — proactively, without being asked. The founder explicitly requested this.

**Why:** The founder wants the workspace to always reflect current reality so it stays investor-grade and any session/agent can trust it.

**How to apply:** Update as you go, not batched at the end. Routing:
- New unknown → `QUESTIONS.md`
- Non-obvious learning / "aha" → `INSIGHTS.md`
- Launch-gating choice or status change → `DECISIONS.md` + propagate resolved values into `SPEC.md`
- Question answered → check off in QUESTIONS.md, move lasting takeaway to INSIGHTS.md
- Anything material → refresh `README.md` status snapshot

The full routing rule also lives in the project's `AGENTS.md` (golden rule #5), which is the source of truth for workspace conventions. See [[realshare-overview]].
